# Unicode

世界中の文字を統一的に扱うための文字コード体系

## Unicode とは

```text
A → U+0041
あ → U+3042
😀 → U+1F600
```

- すべての文字に「番号（コードポイント）」を割り当てる仕組み
- 世界中の言語・記号・絵文字を扱える
- 文字コードの共通ルール

## なぜ必要なのか

昔

- ASCII → 英語だけ
- Shift_JIS → 日本語
- EUC → 別の日本語

文字化け地獄

Unicode → **全部まとめて統一**

## 基本概念

### ● コードポイント

```text
U+3042
```

- 各文字に割り当てられた番号
- 「U+」は Unicode を意味する

### ● 文字と番号の対応

```text
A = U+0041
あ = U+3042
😀 = U+1F600
```

## エンコーディング（超重要）

Unicode そのものは「番号」だけ

- 実際の保存形式が必要

### ● UTF-8（最重要）

```text
可変長（1〜4バイト）
```

- 現代 Web の標準
- 英語は 1 バイト
- 日本語は 3 バイト
- 絵文字は 4 バイト

基本これ使えば OK

### ● UTF-16

```text
2または4バイト
```

- JavaScript 内部で使われる

### ● UTF-32

```text
4バイト固定
```

- シンプルだけど容量大きい

## HTML での使い方

### ● 文字コード指定（超重要）

```html
<meta charset="UTF-8" />
```

これがないと文字化けする

### ● Unicode エスケープ

```html
&#12354; → あ
```

## JavaScript での扱い

### ● Unicode エスケープ

```js
"\u3042"; // あ
```

### ● 絵文字（サロゲートペア）

```js
"😀".length; // 2（注意）
```

UTF-16 の仕様で 2 になる
