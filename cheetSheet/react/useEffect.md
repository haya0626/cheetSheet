# useEffect

### useEffect とは？

`useEffect` は **副作用（side effects）** を扱うためのフックです。

「副作用」とは、**レンダー以外の処理**、つまり以下のようなものです。

- データの取得（API 通信など）
- DOM 操作
- タイマーの設定・解除
- イベントリスナーの登録・解除

## 基本構文

```ts
useEffect(() => {
  // 実行したい副作用
  console.log("Effect 実行");

  return () => {
    // クリーンアップ関数（必要なら）
    console.log("Cleanup 実行");
  };
}, [依存値]);
```

## 実行順序について

※親と子のコンポーネント内に useEffect がある場合

**子 → 親の順番で useEffect が実行される**

## 実行タイミングについて

レンダリングの流れを図にすると、以下のようになります。

```
① コンポーネントがレンダーされる（DOMはまだ反映前）
② 画面に描画される（DOM反映完了）
③ useEffect のコールバックが実行される
```

依存配列（`[]`）の内容によってタイミングが変わります

## クリーンアップのタイミング

クリーンアップ関数（`return () => {...}`）は以下のタイミングで実行されます。

| 状況                               | クリーンアップ実行タイミング |
| ---------------------------------- | ---------------------------- |
| 依存値が変わる                     | 新しい Effect が実行される前 |
| コンポーネントがアンマウントされる | コンポーネント破棄直前       |

### 処理フローの図

```
 ┌───────────────────────────────┐
          コンポーネント描画
 └───────────────────────────────┘
                ↓
        (DOM更新が完了)
                ↓
 ┌───────────────────────────────┐
      useEffect コールバック実行　
 └───────────────────────────────┘
                ↓
        (依存値の変更や再レンダー)
                ↓
 ┌───────────────────────────────┐
  前回の useEffect の Cleanup 実行
 └───────────────────────────────┘
                ↓
        (再び Effect 実行)
```
